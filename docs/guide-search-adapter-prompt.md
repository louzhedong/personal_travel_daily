# Guide Search Adapter Prompt

当你为本地攻略 API 新增或维护适配器时，请遵守以下约束：

## 目标

- 从外部来源提取稳定的攻略摘要或正文
- 输出统一的数据结构
- 尽量减少对前端结构的影响

## 适配器位置

- `server/adapters/*.mjs`
- 统一注册入口：`server/adapters/index.mjs`

## 当前服务端行为

- 搜索时会合并缓存、种子数据、静态条目和远程抓取结果
- 拉取正文时优先命中种子和缓存
- 服务端输出统一的搜索结果与正文格式

## 适配器实现要求

1. 保持输出字段完整：标题、摘要、来源、来源链接、目的地标签等。
2. 远程抓取失败时，要让上层能区分是“无结果”还是“来源不可用”。
3. 适配器应尽量避免脆弱的页面结构依赖。
4. 可缓存的正文数据应交给文件存储层管理。

## 新增适配器后的同步工作

- 更新 `/health` 的适配器列表
- 更新 `docs/guide-search-api-contract.md` 中的说明
- 如涉及环境变量，更新 `.env.example` 与 README
