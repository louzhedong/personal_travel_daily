# Design Prompt

为 `Voyage Atlas` 设计或调整界面时，请遵循以下方向：

## 视觉定位

- 清爽、柔和、偏旅行工具感
- 卡片和地图并重，不要做成纯后台表格
- 信息层级明确，但避免过度装饰

## 设计要点

### 地图区

- 地图是主视觉之一
- hover、选中、定位状态要清晰区分
- 不要让旁路信息覆盖地图主任务

### 侧栏与面板

- 时间线、收藏、旅伴管理都属于辅助信息区
- 保持卡片统一风格，但每块信息密度要不同
- 长列表要考虑滚动提示与视觉边界

### 表单与筛选

- 不要滥用加粗
- 下拉框、按钮、chip 需要统一圆角和边框体系
- 文案尽量自然，不使用生硬标签词

### 弹窗

- 高价值但不高频的功能适合弹窗，例如数据备份与恢复
- 弹窗头部结构和关闭方式要统一

## 当前已形成的视觉约束

- 时间线滚动区带上下渐隐
- 数据备份入口放在旅行记录模块，不常驻侧栏
- 筛选控件统一采用轻边框、圆角、不过度加粗的风格

## 输出要求

- 设计说明需要落到组件或样式文件层级
- 如需改动样式，说明应该归属哪个 `src/styles/components/*.css` 文件
