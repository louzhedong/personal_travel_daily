# Design Prompt

你正在为 `旅迹地图 / Voyage Atlas` 设计或修改界面。

请始终遵循当前项目已经建立起来的旅行品牌视觉系统，而不是把页面做成通用工具后台。

## 产品视觉定位

这是一个旅行记录产品原型，视觉上应当体现：

- 地图感
- 路线感
- 旅途感
- 内容感
- 品牌感

不是：

- 后台管理系统
- 强表单系统
- 高密度操作面板

## 设计关键词

- 轻
- 克制
- 柔和
- 品牌化
- 地图语义
- 卡片化

## 当前视觉语言

### 颜色

当前主品牌色系：

- 蓝：路线、地图、探索
- 青：自然、旅途、清新氛围
- 橙：目的地、亮点、行动点缀

应尽量复用现有品牌色，不要新增无语义的随机颜色。

### 形态

- 大圆角
- 柔和阴影
- 轻背景分层
- 尽量避免明显描边
- 能用层次区分，就少用边框

### 动效

- 轻微 hover
- 轻微上浮
- 柔和渐变或点亮感
- 不要夸张跳动
- 动效优先服务“旅途 / 路线 / 地图”语义

## 已有模块的视觉标准

### Hero

- 品牌首页感
- 抽象地图线稿
- 经纬线网格
- 航线点亮

### 统计卡

- 像“目的地卡片”
- 不是普通数据块
- 有图标、轻文案、轻 hover

### 地图模块

- 标题区是品牌化头部
- 说明胶囊与视图切换已做成统一控件组
- 控件层次要轻，不要边框套边框
- 地图内容区本身应保持干净

### 攻略搜索面板

- 这是“旅途灵感 / 目的地信息”入口，不是搜索后台
- 应延续现有抽屉 / 侧边详情面板语义
- 搜索框、筛选 chips、结果卡片都应偏内容化呈现
- 正文片段区应像“旅行摘要”而不是文档管理器

### 旅伴管理

- 更像“旅伴 / 同行人”管理，而不是用户后台
- 头像徽章、在线状态、颜色主题名都偏内容化呈现

## 设计约束

### 避免

- 过多边框
- 深色硬描边
- 多层控件套框
- 过大的 placeholder
- 强功能态图标导致界面工具化
- 一切“太像后台”的排版和控件处理

### 倾向

- 用背景层次替代边框
- 用轻高亮替代重描边
- 用品牌图标替代生硬功能图标
- 用内容摘要替代纯数字 / 纯状态提示
- 用目的地、来源、标签、摘要建立内容层次，减少工具栏感

## 地图相关设计要求

如果改地图模块，请遵循：

- 新控件不要压住地图主体
- 新视觉层不要干扰区域识别
- hover 提示优先简洁、信息有效
- 轨迹、箭头、路线等元素要轻，不要压过地图本身
- 放大缩小时视觉要稳定

## 旅途轨迹设计要求

如果设计轨迹线：

- 默认不应喧宾夺主
- 应该服务“时间顺序”和“路线感”
- 可以用：
  - 轻虚线
  - 小箭头
  - 柔和颜色
  - 轻 hover 高亮
- 不要做成很厚、很亮、很花哨的导航线

## 当你输出设计方案时

请尽量包含：

1. 你想强化的产品语义
   - 地图感 / 旅途感 / 目的地感 / 品牌感
2. 你打算怎么减弱后台感
3. 你是否引入了新的边框、阴影、颜色、动效
4. 为什么这些变化仍然符合当前项目视觉系统

如果任务与攻略搜索相关，请额外说明：

1. 你如何让搜索面板更像旅行内容入口，而不是检索工具
2. 搜索框、筛选 chips、结果卡片、正文片段区的层次关系
3. 是否保持与 `MarkerDetailPanel` 相近的抽屉式语言

## 一句话原则

这个项目的设计目标是：

**做成一个轻量、克制、带旅行品牌感的地图记录产品，而不是一个通用数据管理后台。**
