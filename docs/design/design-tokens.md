# 视觉 Token 说明

当前项目尚未引入独立设计系统包，但已经形成一套稳定的样式分层与 token 使用习惯。

## 文件分层

- `src/styles/base.css`: 全局变量、reset、基础字体和元素样式
- `src/styles/layout.css`: 页面壳层、卡片、弹窗和通用布局
- `src/styles/home.css`: 首页头部与欢迎区视觉
- `src/styles/responsive.css`: 响应式断点下的覆盖规则
- `src/styles/components/*.css`: 模块级样式

## Token 使用原则

### 颜色

- 页面主色用于地图强调、按钮高亮、激活状态
- 信息卡、筛选 chips、标签和状态徽标应复用同一套主/辅色阶
- 不要直接在组件中写大量硬编码色值，优先放回样式文件中的变量区统一维护

### 圆角

- 大卡片、面板与弹窗使用较大圆角
- 表单控件与列表条目使用中等圆角
- 小标签和按钮使用较小但仍明显的圆角，保持柔和观感

### 阴影与边框

- 外层卡片采用轻阴影 + 低对比边框
- 激活态使用主色边框或主色投影
- 不建议在同一组件上同时堆叠过多阴影层级

### 间距

- 页面块与块之间优先保持 20px 左右节奏
- 面板内部常用 12px / 16px / 20px 三档
- 相邻文本和徽标尽量沿用同一排版节奏

### 字体

- 正文保持中性、清晰、不过分加粗
- 筛选控件和下拉框默认不加粗，避免和标题抢视觉层级
- 标题、说明、副标题之间要保持明显但克制的层级差异

## 当前已落地的组件视觉约束

- 时间线列表隐藏滚动条，但保留滚动能力
- 时间线上下边缘使用渐隐遮罩提示可继续滚动
- 数据备份与恢复采用弹窗承载，不再占据常驻侧栏空间
- 列表筛选控件采用统一圆角表单风格

## 后续建议

- 如果继续扩展页面，优先在 `base.css` 中提取更显式的 CSS 变量命名
- 当某个业务模块样式继续增长时，优先细分 `components/*.css`
